# Review Tag | Frontend

Review Tag는 **영화·콘텐츠 기반 회원제 리뷰 플랫폼**입니다.  
리뷰/별점, 가격 평가, 신뢰도 시스템, 자유게시판/댓글, 콘텐츠 퀴즈, 포인트 기능을 한 서비스 안에 담아 사용자의 참여가 자연스럽게 이어지도록 설계했습니다.

특히 이 프로젝트는 단순히 리뷰를 남기는 데서 끝나는 구조가 아니라, **활동에 대한 보상과 소비 경험이 자연스럽게 이어지는 서비스 흐름**을 목표로 했습니다.  
저는 그중에서도 **포인트 관련 기능 프론트엔드 구현**을 맡아 출석, 퀘스트, 상점, 인벤토리, 이력, 관리자 화면이 하나의 사용자 경험으로 이어지도록 작업했습니다.

---

## 프로젝트 개요

- 영화·콘텐츠 기반 회원제 리뷰 플랫폼
- 리뷰/별점과 함께 콘텐츠 가치를 가격으로 표현하는 평가 방식 제공
- 신뢰도 시스템, 자유게시판/댓글, 퀴즈를 통한 커뮤니티형 서비스 구조
- 포인트 적립과 소비를 연결해 재방문과 참여를 유도하는 보상 흐름 설계

---

## 담당 역할

팀 프로젝트에서 **포인트 관련 기능 프론트엔드 구현**을 중심으로 작업했습니다.

- 포인트 메인 대시보드 구성
- 출석체크 및 출석 캘린더 화면 연결
- 일일 퀘스트, 데일리 퀴즈, 룰렛 UI 구현
- 포인트 상점, 위시리스트, 선물하기 흐름 구현
- 인벤토리 조회, 사용, 장착, 해제, 환불 화면 구현
- 포인트 이력 및 관리자 운영 화면 연결
- 인증 처리, 재조회 패턴, 화면 상태 동기화 정리

제가 맡은 범위의 핵심은 화면을 각각 만드는 것보다, **적립과 소비 흐름이 사용자 입장에서 끊기지 않게 보여주는 것**이었습니다.

## 구현 포인트

### 1. 인증 흐름을 한 군데로 모았습니다.
페이지마다 인증 처리가 달라지지 않도록 axios instance를 공통으로 두고, interceptor에서 토큰 주입과 재요청 흐름을 함께 관리했습니다.

### 2. 포인트 관련 값은 성공 후 재조회로 맞췄습니다.
잔액, 재고, 인벤토리 수량, 퀘스트 보상 여부처럼 서로 연결된 값은 화면에서 임의로 계산하지 않고, 성공 응답 이후 다시 조회하는 방식으로 정합성을 맞췄습니다.

### 3. 사용자 흐름이 끊기지 않도록 화면을 연결했습니다.
출석, 퀘스트, 퀴즈, 룰렛, 상점, 인벤토리가 각각 따로 보이지 않도록, 사용자가 행동한 결과가 다음 화면 변화로 자연스럽게 이어지도록 구성했습니다.

### 4. 관리자 화면도 같은 기준으로 정리했습니다.
사용자 화면뿐 아니라 관리자 포인트 조정, 상품 관리, 자산 조회 화면도 비슷한 패턴으로 연결해 운영 흐름까지 함께 고려했습니다.
