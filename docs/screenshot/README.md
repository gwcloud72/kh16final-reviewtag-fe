# Review Tag Frontend | 화면 상세 설명
이 문서는 포인트 프론트의 전체 화면을 한눈에 살펴볼 수 있도록, 각 주요 화면을 실제 스크린샷과 함께 꼼꼼히 정리한 자료입니다. 루트 README에서는 대표적인 화면 위주로만 간략하게 소개했다면, 이 문서에서는 사용자가 접할 수 있는 거의 모든 장면을 빠짐없이 캡처하여 담았습니다.

## 읽는 순서 안내

1. 메인 대시보드 및 출석 흐름
2. 상점과 인벤토리 주요 동선
3. 포인트 이력 내역 확인
4. 관리자 운영 화면 살펴보기

---

## 01. 메인 대시보드

![메인 대시보드](01-main-dashboard.png)

### 화면 설명
포인트 서비스의 첫 진입점이 되는 대시보드 화면입니다. 프로필 카드, 당일 할 수 있는 주요 활동, 그리고 상단 탭 구조까지 한눈에 들어올 수 있도록 배치했습니다.

### 주요 기능
- 현재 보유 포인트와 꾸미기 상태 즉시 확인
- 일일 퀘스트 진입 및 출석 캘린더 확인
- 상점, 인벤토리, 이력 정보로 빠른 이동

### 담당 역할 및 구현 포인트
이 화면에서는 각각 분리된 출석, 퀘스트, 상점, 인벤토리 기능들이 자연스럽게 이어지도록 구조를 설계하는 데 중점을 뒀습니다. 포인트 획득, 아이템 구매처럼 한 탭에서 발생한 변화가 다른 탭에도 실시간으로 어색함 없이 반영될 수 있도록, PointMain 컴포넌트에 전체 흐름의 기준축을 세웠습니다.

---

## 02. 출석

![출석 화면](02-attendance.png)

### 화면 설명
사용자의 당일 출석 여부와 일자별 출석 도장을 확인할 수 있는 공간입니다.

### 주요 기능
- 오늘 출석 체크 및 버튼 노출
- 완료된 출석 표시와 달력 내 스탬프 반영
- 연속 출석 흐름도 시각적 안내

### 담당 역할 및 구현 포인트
출석은 단순히 버튼만 누르는 구조처럼 보이지만, 실제로는 버튼 상태, 달력 표시, 카드 포인트 등 관련 화면 요소들이 함께 바뀌어야 더욱 자연스럽게 느껴질 수 있습니다. 때문에 출석 성공 후에는 관련된 데이터와 UI들이 한꺼번에 업데이트되도록 설계했습니다.

---

## 03. 상점

![상점 화면](03-store.png)

### 화면 설명
포인트로 아이템을 직접 구매하거나, 타인에게 선물할 수 있는 상점 화면입니다.

### 주요 기능
- 다양한 상품 목록 확인 및 필터/검색 기능
- 위시리스트 추가 및 보유 여부 표시
- 구매 및 선물 기능 제공

### 담당 역할 및 구현 포인트
상점은 단순하게 상품 목록만 보여준다고 끝나는 화면이 아닙니다. 사용자가 이미 보유한 상품인지, 또는 찜한 상품인지 한눈에 알아볼 수 있도록 시각적으로 구분했고, 구매 후에는 즉시 관련 상태와 값이 반영될 수 있도록 로직을 연결했습니다.

---

## 04. 인벤토리

![인벤토리 화면](04-inventory.png)

### 화면 설명
사용자가 구매한 아이템을 실제로 사용할 수 있도록 구성된 곳입니다.

### 주요 기능
- 아이템별 사용/장착/해제 기능
- 환불, 버리기, 랜덤 아이템 사용 등 부가 기능 제공
- 닉네임 변경권 및 꾸미기 아이템의 별도 처리

### 담당 역할 및 구현 포인트
같은 버튼처럼 보여도 각 아이템별로 동작 방식이 다르기 때문에, 타입별 분기를 먼저 세분화했습니다. 그리고 마지막에는 모든 동작이 공통된 refresh 기준으로 묶여, 사용성의 일관성을 확보했습니다.

---

## 05. 포인트 이력

![포인트 이력](05-history.png)

### 화면 설명
포인트가 어디서 적립되고, 어떤 이유로 차감됐는지 명확하게 확인할 수 있는 기록 화면입니다.

### 주요 기능
- 적립/차감 내역 세부 조회
- 발생 사유와 금액 안내
- 최근 활동 흐름 한눈에 보기

### 담당 역할 및 구현 포인트
출석, 퀘스트, 구매, 환불 등 다양한 기능에서 발생한 포인트 변동 이력이 하나의 화면에 모이도록 흐름을 연결했습니다. 덕분에 사용자는 복잡할 것 없이, 자신이 보유한 포인트의 전체 사용 내역을 한 번에 파악할 수 있습니다.

---

## 06. 관리자 포인트 화면

![관리자 포인트](06-admin-point.png)

### 화면 설명
관리자가 회원의 포인트를 지급하거나 차감하며, 처리 이력을 관리하는 운영용 화면입니다.

### 주요 기능
- 회원 검색 기능
- 포인트 지급 및 회수 처리
- 관리 내역 상세 확인

### 담당 역할 및 구현 포인트
운영 화면이라 하더라도 사용자 화면과 완전히 동떨어지게 보이지 않도록, 포인트 수치를 변경하는 흐름이 결국 같은 맥락 안에서 연결되도록 맞췄습니다.
이처럼 각각의 화면은 독립적으로 기능을 수행하는 동시에, 서로 자연스럽게 연결되어 사용자가 서비스 이용 과정에서 이질감 없이 흐름을 따라갈 수 있도록 신경 써서 설계하고 구현했습니다.
---

## 07. 관리자 상점 화면

![관리자 상점](07-admin-store.png)

### 화면 설명
운영자가 직접 상점에 등록된 상품을 추가하거나 수정하고, 필요시 삭제하는 관리자 전용 페이지입니다.

### 주요 기능
- 상품 목록 전체 관리
- 재고, 가격, 이미지 등 상품 정보 수정
- 신규 상품의 간편 등록

### 담당 역할 및 구현 의도
단순히 사용자용 상점만 연결하고 끝내는 것이 아니라, 실제 운영환경을 염두에 두고 관리자 화면까지 체계적으로 구축했습니다. 운영자가 복잡한 상품 관리 업무도 손쉽게 처리할 수 있도록 각각의 기능을 한눈에 파악할 수 있게 설계한 점이 특징입니다.

---

## 08. 관리자 자산 화면

![관리자 자산](08-admin-inventory.png)

### 화면 설명
회원들이 보유하고 있는 자산과 각 인벤토리 내역을 운영자 시점에서 효율적으로 확인할 수 있도록 구성한 화면입니다.

### 주요 기능
- 회원별 보유 아이템 확인 및 검색
- 자산 현황 전체 조회
- 운영자가 필요로 하는 맞춤형 검색 및 확인

### 담당 역할 및 구현 의도
인벤토리 기능을 단순히 사용자 화면에만 제한하지 않고, 운영에서도 동일한 기준과 흐름으로 확인할 수 있도록 만들었습니다. 이를 통해 실시간으로 회원 자산을 빠짐없이 관리할 수 있는 기반을 마련했습니다.

---

## 핵심 정리

포인트 시스템의 프론트엔드 개발에서 가장 중점을 둔 것은 ‘화면 개수’가 아니라, 여러 컴포넌트 곳곳에 흩어진 동일 포인트 값을 어떻게 항상 일치하게 보여줄지에 있었습니다. 이 문서에 첨부한 스크린샷들 역시, 제가 담당한 기능들이 각 화면 내부에서 어떻게 분산되지 않고 유기적으로 연결되어 있는지 직관적으로 확인할 수 있게 구성했습니다. 실제 운영을 고려한 설계와 일관된 데이터 연동이 이 프로젝트에서 가장 중요했던 부분입니다.
