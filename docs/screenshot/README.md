# Review Tag Frontend | 화면 상세 설명

이 문서는 포인트 프론트 화면을 스크린샷 기준으로 빠르게 훑어보기 위해 작성되었습니다. 루트 README에서는 대표 화면만 보여주지만, 여기서는 사용자가 마주하는 거의 모든 화면의 캡처본을 빠짐없이 정리했습니다.

## 읽는 순서

1. 메인 대시보드와 출석 흐름
2. 상점과 인벤토리 흐름
3. 포인트 이력 확인
4. 관리자 운영 화면

---

## 01. 메인 대시보드

![메인 대시보드](01-main-dashboard.png)

### 화면 설명
포인트 기능 진입 화면입니다. 프로필 카드, 오늘 할 수 있는 행동, 탭 이동 구조가 한 번에 보입니다.

### 여기서 보이는 기능
- 현재 보유 포인트와 꾸미기 상태 표시
- 일일 퀘스트 진입
- 출석 캘린더 확인
- 상점 / 인벤토리 / 이력 탭 이동

### 이 화면에서 담당한 기능
이 화면에서는 출석, 퀘스트, 상점, 인벤토리 등 여러 기능들이 서로 단절된 느낌 없이 자연스럽게 이어지도록 묶는 데 가장 신경을 썼습니다. 사용자가 한 탭에서 한 행동의 결과(포인트 획득, 아이템 구매 등)가 다른 탭에도 어색함 없이 즉각 반영될 수 있도록, PointMain 컴포넌트에서 전체적인 상태와 흐름의 기준을 확실하게 잡았습니다

---

## 02. 출석

![출석 화면](02-attendance.png)

### 화면 설명
오늘 출석 여부와 출석 도장을 확인하는 화면입니다.

### 여기서 보이는 기능
- 출석 버튼 노출
- 오늘 출석 완료 여부 표시
- 달력 스탬프 반영
- 연속 출석 흐름 확인

### 이 화면에서 담당한 기능
출석은 버튼 하나처럼 보이지만, 실제로는 버튼 상태와 달력, 카드 포인트가 같이 바뀌어야 자연스럽습니다. 그래서 출석 성공 뒤에는 관련 화면이 함께 다시 읽히도록 구성했습니다.

---

## 03. 상점

![상점 화면](03-store.png)

### 화면 설명
포인트로 아이템을 구매하거나 선물하는 화면입니다.

### 여기서 보이는 기능
- 상품 목록 조회
- 검색과 필터
- 위시리스트 표시
- 구매 / 선물하기
- 보유 여부 확인

### 이 화면에서 담당한 기능
상점은 목록만 보여주면 끝나는 화면이 아니라, **내가 이미 가진 상품인지**, **찜해둔 상품인지**, **구매 뒤 어떤 값이 바뀌는지**를 같이 보여줘야 했습니다. 그래서 목록, 보유 상태, 위시 상태를 함께 연결했습니다.

---

## 04. 인벤토리

![인벤토리 화면](04-inventory.png)

### 화면 설명
구매한 아이템을 실제로 사용하는 화면입니다.

### 여기서 보이는 기능
- 아이템별 사용 버튼
- 장착 / 해제
- 환불 / 버리기
- 랜덤 아이템 사용
- 닉네임 변경권, 꾸미기 아이템 처리

### 이 화면에서 담당한 기능
버튼 모양은 비슷해도 아이템 타입마다 동작 방식이 달라서, 타입별 분기를 먼저 정리하고 마지막에는 공통 refresh 기준으로 묶었습니다.

---

## 05. 포인트 이력

![포인트 이력](05-history.png)

### 화면 설명
포인트가 어디서 적립되고 차감됐는지 확인하는 화면입니다.

### 여기서 보이는 기능
- 적립 / 차감 내역 조회
- 사유와 금액 확인
- 최근 활동 흐름 확인

### 이 화면에서 담당한 기능
출석, 퀘스트, 구매, 환불처럼 서로 다른 기능에서 바뀐 값이 한 이력 화면으로 모여야 했습니다. 그래서 사용자 입장에서도 포인트 흐름을 한 번에 이해할 수 있게 연결했습니다.

---

## 06. 관리자 포인트 화면

![관리자 포인트](06-admin-point.png)

### 화면 설명
관리자가 회원 포인트를 지급하거나 회수하는 운영 화면입니다.

### 여기서 보이는 기능
- 회원 검색
- 포인트 지급 / 차감
- 처리 이력 확인

### 이 화면에서 담당한 기능
사용자 화면과 운영 화면이 완전히 다른 구조처럼 보이지 않게, 포인트 값이 바뀌는 결과가 결국 같은 흐름 안에서 보이도록 맞췄습니다.

---

## 07. 관리자 상점 화면

![관리자 상점](07-admin-store.png)

### 화면 설명
운영자가 상점 상품을 등록, 수정, 삭제하는 화면입니다.

### 여기서 보이는 기능
- 상품 목록 관리
- 재고 / 가격 / 이미지 수정
- 신규 상품 등록

### 이 화면에서 담당한 기능
사용자용 상점만 연결하면 끝나는 프로젝트처럼 보이지 않게, 실제 운영까지 고려한 화면으로 정리했습니다.

---

## 08. 관리자 자산 화면

![관리자 자산](08-admin-inventory.png)

### 화면 설명
회원이 보유한 자산과 인벤토리를 운영 관점에서 확인하는 화면입니다.

### 여기서 보이는 기능
- 회원별 보유 아이템 확인
- 자산 현황 조회
- 운영용 검색 / 확인 기능

### 이 화면에서 담당한 기능
인벤토리 기능을 사용자 화면에만 두지 않고, 운영 화면에서도 같은 기준으로 확인할 수 있게 맞췄습니다.

---

## 짧게 정리하면

포인트 관련 프론트엔드 작업의 핵심은 화면의 개수가 아니라, 여러 컴포넌트에 흩어져 있는 동일한 포인트 값을 어떻게 항상 일치하게 보여줄 것인가에 있었습니다. 이 문서의 스크린샷들은 제가 맡은 기능들이 각 화면에서 파편화되지 않고, 어떻게 유기적으로 연결되며 동작하는지 빠르게 파악할 수 있도록 구성했습니다.
